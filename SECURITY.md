# Security policy

## Reporting

Do not open public issues for vulnerabilities. Use a private GitHub security advisory at:

https://github.com/stevancarlon/omni-mobile/security/advisories/new

Include affected versions, impact, reproduction steps using synthetic data, and a proposed fix if available. Do not access other people's data or test against accounts you do not own.

## Scope

High-priority reports include unauthorized Accessibility actions, sensitive-app guard bypasses, exported-component abuse, token disclosure, insecure update/signing behavior, screenshot leakage, and model-output paths that bypass action validation.

Only APKs attached to releases in the official repository and signed by the documented official certificate should be treated as official. Forks and third-party builds are operated by their distributors.

Omni is alpha software and currently receives security fixes on a best-effort basis. No version is designated as long-term supported.
